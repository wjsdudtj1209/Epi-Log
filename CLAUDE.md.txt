# 프로젝트 규칙 문서

## 1. 프로젝트 목적
- 신규 앱 서비스를 소개하고 사용자를 모으기 위한 랜딩 페이지 웹사이트 제작

## 2. 개발 환경 및 기술 스택
- 아직 개발 환경이 세팅되지 않았으니, 클로드가 판단하여 가장 빠르고 트렌디한 프론트엔드 환경(예: Vite + React + Tailwind CSS)을 직접 설치하고 구축해 줄 것.
- 화려한 웹 인터랙션(애니메이션 효과)을 위해 `Framer Motion` 라이브러리를 적극 사용할 것.

## 3. 개발 방식 규칙
- 모든 코드는 초보자가 이해하기 쉽게 파일별로 역할을 나누어 작성할 것.
- 디자인은 모바일과 PC 화면 모두에서 예쁘게 보이는 '반응형 디자인'으로 만들 것.

## 4. 진행 순서
1. `docs/` 폴더에 있는 기획서를 먼저 읽고 앱 서비스의 컨셉을 파악한다.
2. 웹사이트의 전체적인 색상(테마)과 폰트를 제안하고, 개발 프로젝트 뼈대를 생성한다.
3. 기획서 내용에 맞춰 섹션별로 웹사이트를 순서대로 구현한다.
